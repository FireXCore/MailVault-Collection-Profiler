# Governance

The project is currently maintainer-led during the `0.x` architecture and compatibility phase.

Maintainers control:

- source compatibility contracts;
- security and privacy invariants;
- profiler schema migrations;
- release acceptance gates;
- roadmap slice boundaries;
- public release artifacts.

Significant architecture changes require a documented proposal in an issue or pull request before
implementation. Compatibility, source mutation or evidence semantics cannot be changed through an
unreviewed refactor.
