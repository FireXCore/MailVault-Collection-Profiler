# Support

## Usage and build questions

Use GitHub Discussions only when it is enabled. Otherwise open an issue using the appropriate form:

- bug report;
- feature request;
- MailVault compatibility report.

Before opening an issue:

1. read the getting-started and troubleshooting guides;
2. run the current quality gate when reporting a source-build problem;
3. reproduce with a synthetic or sanitized fixture;
4. remove private paths, filenames, message subjects, domains and hashes.

## Unsupported support material

Do not upload:

- MailVault or profiler databases;
- EML files or attachments;
- credentials;
- unredacted runtime evidence;
- screenshots containing private business information.

## Security problems

Do not open a public issue. Follow [SECURITY.md](SECURITY.md).
